## 🎯 推荐执行方式

### 方式1: 一次性执行（推荐有高性能GPU）

```bash
# 一次性分析所有日期
python scripts/batch_analyze_all_dates.py --base_dir /mnt/f
```

**预计时间**: 30分钟 - 1小时（高性能GPU）

### 方式2: 分批执行（推荐中低性能GPU或担心时间过长）

#### 分批1: 先分析2025年的日期（4个日期）
```bash
python scripts/batch_analyze_all_dates.py \
    --base_dir /mnt/f \
    --date_pattern "2025.*"
```
**预计时间**: 15-40分钟（高性能GPU）

#### 分批2: 再分析24年和25年的日期（4个日期）
```bash
python scripts/batch_analyze_all_dates.py \
    --base_dir /mnt/f \
    --date_pattern "24.*|25.*" \
    --skip_existing
```
**预计时间**: 15-40分钟（高性能GPU）

### 方式3: 逐个日期执行（最灵活）

```bash
# 分析单个日期
python scripts/batch_analyze_by_date.py \
    --base_dir /mnt/f/2025.09.06/extracted_frames
```

## 💡 优化建议

1. **使用GPU**: 确保CUDA可用，速度提升10-20倍
2. **分批执行**: 如果担心时间过长，可以分批执行
3. **后台执行**: 可以使用nohup或screen在后台运行
   ```bash
   nohup python scripts/batch_analyze_all_dates.py --base_dir /mnt/f > analysis.log 2>&1 &
   ```
4. **中断恢复**: 如果中断，使用 `--skip_existing` 继续
   ```bash
   python scripts/batch_analyze_all_dates.py --base_dir /mnt/f --skip_existing
   ```

## ⚠️ 注意事项

- **GPU内存**: 确保GPU内存足够（至少4GB）
- **磁盘I/O**: 大量图片读写可能影响速度
- **网络存储**: 如果数据在网络存储上，速度会更慢
- **系统负载**: 其他程序可能影响速度
