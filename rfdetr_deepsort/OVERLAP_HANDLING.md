# 重叠处理机制说明

## 🎯 问题描述

当多个目标（如羊群）重叠时，可能会出现以下问题：

1. **ID 相互影响**：重叠的检测框可能被误判为未匹配，导致创建新ID
2. **ID 快速增长**：每次重叠都可能产生新的ID，导致ID数量不稳定
3. **匹配不稳定**：重叠时匹配可能不稳定，导致ID交换

## ✨ 解决方案

### 1. 重叠检测 (`detect_overlapping_detections`)

**功能**：检测当前帧中哪些检测框相互重叠

**算法**：
- 计算所有检测框之间的 IoU
- 如果 IoU > `overlap_threshold`（默认0.5），认为重叠
- 将重叠的检测框分组

**输出**：
- `overlap_groups`：重叠组列表，每个组包含重叠的检测索引
- `overlap_mask`：布尔数组，标记哪些检测处于重叠状态

### 2. 重叠时更严格的匹配阈值

**策略**：
- 正常情况：使用 `iou_threshold`（默认0.3）
- 重叠情况：使用 `strict_overlap_threshold`（默认0.6）

**效果**：
- 重叠的检测需要更高的IoU才能匹配到轨迹
- 减少误匹配，提高匹配准确性

### 3. 重叠时保守的新轨迹创建

**策略**：
当未匹配的检测处于重叠状态时：
1. 检查是否与现有已确认轨迹有较高IoU（>0.4）
2. 如果有，不创建新轨迹（可能是误判的未匹配）
3. 如果没有，才创建新轨迹

**效果**：
- 避免重叠时创建重复的ID
- 减少ID数量的快速增长
- 提高ID稳定性

## 📊 工作流程

```
检测结果
    ↓
检测重叠 (detect_overlapping_detections)
    ↓
标记重叠检测 (overlap_mask)
    ↓
计算成本矩阵
    ├─ 正常检测：使用 iou_threshold (0.3)
    └─ 重叠检测：使用 strict_overlap_threshold (0.6)
    ↓
匈牙利算法匹配
    ↓
处理未匹配检测
    ├─ 正常检测：直接创建新轨迹
    └─ 重叠检测：检查是否与现有轨迹有高IoU
        ├─ 有高IoU：不创建（可能是误判）
        └─ 无高IoU：创建新轨迹
```

## ⚙️ 参数说明

### `overlap_threshold` (默认: 0.5)
- **作用**：判断两个检测框是否重叠的阈值
- **调整建议**：
  - 如果重叠检测太敏感（误判），**降低**此值（如0.4）
  - 如果重叠检测不够敏感（漏检），**提高**此值（如0.6）

### `strict_overlap_threshold` (默认: 0.6)
- **作用**：重叠时使用的更严格的IoU匹配阈值
- **调整建议**：
  - 如果重叠时匹配太严格（导致丢失），**降低**此值（如0.5）
  - 如果重叠时仍有误匹配，**提高**此值（如0.7）

### 重叠检查阈值 (代码中: 0.4)
- **作用**：判断未匹配的重叠检测是否应该创建新轨迹
- **调整建议**：
  - 如果仍有重复ID创建，**提高**此值（如0.5）
  - 如果新目标无法创建，**降低**此值（如0.3）

## 🐛 调试建议

### 如果ID仍然快速增长

1. **提高重叠检测阈值**：
   ```python
   overlap_threshold=0.6  # 从0.5提高到0.6
   ```

2. **提高严格匹配阈值**：
   ```python
   strict_overlap_threshold=0.7  # 从0.6提高到0.7
   ```

3. **提高重叠检查阈值**：
   在代码中修改：
   ```python
   if iou_val > 0.5:  # 从0.4提高到0.5
   ```

### 如果新目标无法创建

1. **降低严格匹配阈值**：
   ```python
   strict_overlap_threshold=0.5  # 从0.6降低到0.5
   ```

2. **降低重叠检查阈值**：
   ```python
   if iou_val > 0.3:  # 从0.4降低到0.3
   ```

## 💡 最佳实践

1. **先使用默认参数测试**
2. **观察ID数量变化**：
   - 如果ID数量持续增长，提高阈值
   - 如果新目标无法跟踪，降低阈值
3. **根据场景调整**：
   - 密集场景：使用更严格的阈值
   - 稀疏场景：可以使用更宽松的阈值
