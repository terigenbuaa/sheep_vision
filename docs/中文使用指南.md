# RF-DETR 中文使用指南

本文档汇总了 RF-DETR 的运行（Run）和训练（Train）相关文档的中文说明。

---

## 📖 第一部分：运行模型（Run）

### 一、目标检测模型运行

RF-DETR 支持四种目标检测模型：Nano、Small、Medium、Large。这些模型基于 Microsoft COCO 数据集训练。

#### 1. 在单张图片上运行

**使用 Inference 包（推荐）：**

```python
import os
import supervision as sv
from inference import get_model
from PIL import Image
from io import BytesIO
import requests

url = "https://media.roboflow.com/dog.jpeg"
image = Image.open(BytesIO(requests.get(url).content))

model = get_model("rfdetr-base")  # 可替换为 rfdetr-nano/small/medium/large

predictions = model.infer(image, confidence=0.5)[0]
detections = sv.Detections.from_inference(predictions)
labels = [prediction.class_name for prediction in predictions.predictions]

annotated_image = image.copy()
annotated_image = sv.BoxAnnotator(color=sv.ColorPalette.ROBOFLOW).annotate(annotated_image, detections)
annotated_image = sv.LabelAnnotator(color=sv.ColorPalette.ROBOFLOW).annotate(annotated_image, detections, labels)

sv.plot_image(annotated_image)
```

**使用本地 rfdetr 包：**

```python
from rfdetr import RFDETRBase
from rfdetr.util.coco_classes import COCO_CLASSES

model = RFDETRBase()
detections = model.predict(image, threshold=0.5)
```

#### 2. 在视频文件上运行

```python
import supervision as sv
from rfdetr import RFDETRBase
from rfdetr.util.coco_classes import COCO_CLASSES

model = RFDETRBase()

def callback(frame, index):
    detections = model.predict(frame[:, :, ::-1], threshold=0.5)
    
    labels = [
        f"{COCO_CLASSES[class_id]} {confidence:.2f}"
        for class_id, confidence
        in zip(detections.class_id, detections.confidence)
    ]

    annotated_frame = frame.copy()
    annotated_frame = sv.BoxAnnotator().annotate(annotated_frame, detections)
    annotated_frame = sv.LabelAnnotator().annotate(annotated_frame, detections, labels)
    return annotated_frame

sv.process_video(
    source_path="输入视频路径",
    target_path="输出视频路径",
    callback=callback
)
```

#### 3. 在摄像头流上运行

```python
import cv2
import supervision as sv
from rfdetr import RFDETRBase
from rfdetr.util.coco_classes import COCO_CLASSES

model = RFDETRBase()

cap = cv2.VideoCapture(0)  # 0 表示默认摄像头
while True:
    success, frame = cap.read()
    if not success:
        break

    detections = model.predict(frame[:, :, ::-1], threshold=0.5)
    
    labels = [
        f"{COCO_CLASSES[class_id]} {confidence:.2f}"
        for class_id, confidence
        in zip(detections.class_id, detections.confidence)
    ]

    annotated_frame = frame.copy()
    annotated_frame = sv.BoxAnnotator().annotate(annotated_frame, detections)
    annotated_frame = sv.LabelAnnotator().annotate(annotated_frame, detections, labels)

    cv2.imshow("Webcam", annotated_frame)

    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```

#### 5. 批量推理

可以一次性处理多张图片，提高效率：

```python
from rfdetr import RFDETRBase

model = RFDETRBase()

images = [image1, image2, image3]  # 图片列表
detections_list = model.predict(images, threshold=0.5)  # 返回检测结果列表

for image, detections in zip(images, detections_list):
    # 处理每张图片的检测结果
    pass
```

**支持的模型：**
- `rfdetr-nano` - 最小最快的模型
- `rfdetr-small` - 小型模型
- `rfdetr-medium` - 中型模型（推荐）
- `rfdetr-large` - 大型模型

### 二、实例分割模型运行

RF-DETR-Seg (Preview) 是实例分割模型，可以生成分割掩码。

#### 1. 在单张图片上运行

**使用 Inference 包：**

```python
from inference import get_model

model = get_model("rfdetr-seg-preview")  # 注意：使用 seg-preview 模型
predictions = model.infer(image, confidence=0.5)[0]
detections = sv.Detections.from_inference(predictions)

# 使用 MaskAnnotator 而不是 BoxAnnotator
annotated_image = sv.MaskAnnotator(color=sv.ColorPalette.ROBOFLOW).annotate(annotated_image, detections)
annotated_image = sv.LabelAnnotator(color=sv.ColorPalette.ROBOFLOW).annotate(annotated_image, detections, labels)
```

**使用本地包：**

```python
from rfdetr import RFDETRSegPreview

model = RFDETRSegPreview()
detections = model.predict(image, threshold=0.5)
```

#### 2. 在视频/摄像头/RTSP 流上运行

代码结构与目标检测类似，只需：
- 使用 `RFDETRSegPreview()` 替代 `RFDETRBase()`
- 使用 `sv.MaskAnnotator()` 替代 `sv.BoxAnnotator()`

#### 3. 批量推理

同样支持批量处理多张图片。

## 🎓 第二部分：训练模型（Train）

### 一、数据集准备

#### 数据集格式要求

RF-DETR 要求数据集为 **COCO 格式**，目录结构如下：

```
dataset/
├── train/
│   ├── _annotations.coco.json  # 训练集标注文件
│   ├── image1.jpg
│   ├── image2.jpg
│   └── ...
├── valid/
│   ├── _annotations.coco.json  # 验证集标注文件
│   ├── image1.jpg
│   └── ...
└── test/
    ├── _annotations.coco.json   # 测试集标注文件
    ├── image1.jpg
    └── ...
```

**重要提示：**
- 每个子目录（train/valid/test）都需要有自己的 `_annotations.coco.json` 文件
- 如果是分割模型，COCO JSON 中需要包含 `segmentation` 字段（多边形坐标）

**数据集创建工具：**
- 使用 [Roboflow](https://roboflow.com/annotate) 创建或转换数据集
- 可以从 YOLO 格式转换为 COCO 格式
- 可以在 [Roboflow Universe](https://universe.roboflow.com/) 查找预标注数据集

---

### 二、开始训练

#### 1. 目标检测模型训练

```python
from rfdetr import RFDETRBase

model = RFDETRBase()

model.train(
    dataset_dir="数据集路径",
    epochs=100,              # 训练轮数
    batch_size=4,            # 批次大小
    grad_accum_steps=4,      # 梯度累积步数
    lr=1e-4,                 # 学习率
    output_dir="输出路径"    # 保存模型的目录
)
```

**批次大小调整建议：**
- **A100 等大显存 GPU**：`batch_size=16`, `grad_accum_steps=1`（总批次大小 = 16）
- **T4 等小显存 GPU**：`batch_size=4`, `grad_accum_steps=4`（总批次大小 = 16）
- 保持 `batch_size × grad_accum_steps = 16` 以获得最佳效果

#### 2. 实例分割模型训练

```python
from rfdetr import RFDETRSegPreview

model = RFDETRSegPreview()

model.train(
    dataset_dir="数据集路径",
    epochs=100,
    batch_size=4,
    grad_accum_steps=4,
    lr=1e-4,
    output_dir="输出路径"
)
```

### 三、训练参数详解

| 参数 | 说明 |
|------|------|
| `dataset_dir` | COCO 格式数据集路径，包含 train/valid/test 文件夹 |
| `output_dir` | 训练输出目录，保存检查点、日志等 |
| `epochs` | 训练轮数，更多轮数可能提升性能但增加训练时间 |
| `batch_size` | 每次迭代处理的样本数，越大需要越多显存但训练更快 |
| `grad_accum_steps` | 梯度累积步数，用于在显存不足时模拟大批次训练 |
| `lr` | 学习率，控制模型参数更新速度 |
| `lr_encoder` | 编码器专用学习率，用于微调编码器层 |
| `resolution` | 输入图像尺寸，越大精度可能越高但需要更多显存，必须是 56 的倍数 |
| `weight_decay` | L2 正则化系数，防止过拟合 |
| `device` | 运行设备：`cpu` 或 `cuda`，GPU 训练快得多 |
| `use_ema` | 是否使用指数移动平均，通常能提升最终性能 |
| `gradient_checkpointing` | 梯度检查点，降低显存占用但增加训练时间 |
| `checkpoint_interval` | 保存检查点的频率（按轮数） |
| `resume` | 从检查点恢复训练的路径 |
| `tensorboard` | 是否启用 TensorBoard 日志 |
| `wandb` | 是否启用 Weights & Biases 日志 |
| `project` | W&B 项目名称 |
| `run` | W&B 运行名称 |
| `early_stopping` | 是否启用早停，监控验证 mAP |
| `early_stopping_patience` | 早停耐心值，连续多少轮无改进后停止 |
| `early_stopping_min_delta` | 早停最小改进阈值 |
| `early_stopping_use_ema` | 是否使用 EMA 模型进行早停判断 |

### 四、训练检查点

训练过程中会生成多种检查点文件：

#### 检查点类型

1. **`checkpoint.pth`** - 最新检查点，每个 epoch 结束时保存
2. **`checkpoint_<number>.pth`** - 定期检查点，默认每 10 个 epoch 保存一次
3. **`checkpoint_best_ema.pth`** - 最佳 EMA 检查点（基于验证分数）
4. **`checkpoint_best_regular.pth`** - 最佳常规检查点（基于验证分数）
5. **`checkpoint_best_total.pth`** - 最终最佳检查点，仅包含模型权重，用于推理

#### 检查点大小说明

- **训练检查点**：包含模型权重、优化器状态、调度器状态和训练元数据，用于恢复训练
- **评估检查点**：仅包含模型权重（EMA 或常规），用于追踪最佳模型
- **精简检查点**：仅包含最终模型权重，优化用于推理和部署

---

### 五、恢复训练

如果训练中断，可以从检查点恢复：

```python
model.train(
    dataset_dir="数据集路径",
    epochs=100,
    batch_size=4,
    grad_accum_steps=4,
    lr=1e-4,
    output_dir="输出路径",
    resume="checkpoint.pth路径"  # 指定检查点文件
)
```

训练会自动加载模型权重和优化器状态。

---

### 六、早停（Early Stopping）

早停功能可以监控验证 mAP，如果连续多个 epoch 没有改进就停止训练：

```python
model.train(
    dataset_dir="数据集路径",
    epochs=100,
    batch_size=4,
    grad_accum_steps=4,
    lr=1e-4,
    output_dir="输出路径",
    early_stopping=True,                    # 启用早停
    early_stopping_patience=10,              # 10 个 epoch 无改进则停止
    early_stopping_min_delta=0.001,         # 最小改进阈值
    early_stopping_use_ema=False            # 是否使用 EMA 模型判断
)
```

---

### 七、多 GPU 训练

使用 PyTorch 的分布式数据并行（DDP）进行多 GPU 训练：

```bash
python -m torch.distributed.launch --nproc_per_node=8 --use_env main.py
```

**注意事项：**
- 将 `8` 替换为您要使用的 GPU 数量
- 有效批次大小 = `batch_size × GPU数量`
- 需要相应调整 `batch_size` 和 `grad_accum_steps` 以保持总批次大小

## 📝 总结

### 运行模型的关键点：
1. 可以使用 Inference 包或本地 rfdetr 包
2. 支持图片、视频、摄像头、RTSP 流
3. 支持批量推理提高效率
4. 目标检测使用 BoxAnnotator，分割使用 MaskAnnotator

### 训练模型的关键点：
1. 数据集必须是 COCO 格式
2. 合理设置 batch_size 和 grad_accum_steps
3. 使用 TensorBoard 或 W&B 监控训练
4. 利用早停和检查点恢复功能
5. 支持多 GPU 分布式训练
