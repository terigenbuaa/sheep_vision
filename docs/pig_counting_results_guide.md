## 🔬 实验方法论

### 1. 技术栈

| 组件 | 技术 | 说明 |
|------|------|------|
| 目标检测 | RF-DETR | 基于 DINOv2 backbone 的 DETR 变体，针对生猪数据集微调 |
| 多目标跟踪 | ByteTrack / ByteTrack_Temporal | 高性能在线跟踪算法 |
| 计数策略 | 三区域过线计数 | WAIT → ACTIVE → OUT 状态机 |

### 2. 检测模型参数

- **模型架构**: RF-DETR Base (DINOv2-Small Backbone)
- **输入分辨率**: 560×560
- **置信度阈值**: 0.5
- **NMS 阈值**: 0.7

### 3. 跟踪器参数

| 参数 | ByteTrack | ByteTrack_Temporal |
|------|-----------|-------------------|
| track_thresh | 0.5 | 0.5 |
| match_thresh | 0.8 | 0.8 |
| track_buffer | 30 frames | 30 frames |
| 时序增强 | ❌ | ✅ (FDTA inspired) |

### 4. 计数策略详解

采用**三区域状态机**计数，避免边界震荡导致的误计数：

```
        ┌─────────────────────────────────────────────────┐
        │                                                 │
        │            WAIT ZONE (等待区)                    │
        │                  25%                            │
        │                                                 │
        │   目标首次出现在此区域，分配 track_id             │
        │   状态: WAITING                                  │
        │                                                 │
        ├─────────────────────────────────────────────────┤
        │                                                 │
        │            ACTIVE ZONE (活跃区)                  │
        │                  30%                            │
        │                                                 │
        │   目标从 WAIT 进入此区域，状态变为 ACTIVE          │
        │   正常追踪，防止边界抖动误计                       │
        │                                                 │
        ├─────────────────────────────────────────────────┤
        │                                                 │
        │            OUT ZONE (离开区)                     │
        │                  45%                            │
        │                                                 │
        │   目标从 ACTIVE 进入此区域                        │
        │   状态变为 CONFIRMED，计数 +1                    │
        │   该 track_id 不再重复计数                       │
        │                                                 │
        └─────────────────────────────────────────────────┘
                              ↓
                         出口方向 (视频底部)
```

**状态转换规则**:
```
WAITING → ACTIVE (进入活跃区) → CONFIRMED (进入离开区，计数+1)
```

**防误计策略**:
- 轨迹必须经历完整的 WAIT→ACTIVE→OUT 流程才计入
- 生命周期过短的轨迹自动过滤
- 确认后的轨迹不会重复计数

## 📈 跟踪器对比分析

### ByteTrack vs ByteTrack_Temporal (FDTA)

| 指标 | ByteTrack | ByteTrack_Temporal |
|------|-----------|-------------------|
| 总绝对误差 | 27 | 27 |
| 精确匹配数 | 22/35 | 22/35 |
| 误差率 | 1.23% | 1.23% |
| 平均处理时间 | 基准 | 约+3% |

### 结论

两种跟踪器在本数据集上**表现完全一致**：
- ByteTrack_Temporal 的时序增强在当前场景没有带来额外收益
- 推荐使用标准 **ByteTrack**，计算开销更低
- 时序增强可能在更复杂遮挡场景下有优势

## 🎯 输出文件字段说明

### accuracy_report.csv

| 字段 | 类型 | 描述 |
|------|------|------|
| Video | string | 视频文件名 |
| GT | int | Ground Truth 真实标注值 |
| Tracker | string | 使用的跟踪算法 |
| Valid | int | 有效计数 (通过完整状态转换的轨迹数) |
| TotalIDs | int | 总分配的轨迹 ID 数 |
| Error | int | 误差值 = Valid - GT (正=多计，负=漏计) |
| AbsError | int | 绝对误差 = |Error| |
| Accuracy% | float | 准确率 = (Valid / GT) × 100 |

### id_events.csv

| 字段 | 类型 | 描述 |
|------|------|------|
| frame | int | 帧序号 |
| timestamp | float | 时间戳 (秒) |
| track_id | int | 轨迹 ID |
| event | string | 事件类型 |
| center_x | float | 目标中心 X 坐标 |
| center_y | float | 目标中心 Y 坐标 |

**事件类型说明**:
- `enter_wait`: 目标进入等待区，开始追踪
- `enter_active`: 目标进入活跃区
- `confirmed`: 目标进入离开区，确认计数
- `lost`: 轨迹丢失

### trajectory_report.csv

| 字段 | 描述 |
|------|------|
| track_id | 轨迹 ID |
| first_frame | 首次出现帧 |
| last_frame | 最后出现帧 |
| duration_frames | 轨迹持续帧数 |
| final_state | 最终状态 |
| was_counted | 是否被计入最终统计 |

## 🚀 使用脚本

### 单视频分析

```bash
python rfdetr/count_pigs_refined.py \
    --video /path/to/video.mp4 \
    --output /path/to/output \
    --tracker ByteTrack \
    --out_ratio 0.45 \
    --wait_ratio 0.25 \
    --confidence 0.5
```

### 批量分析

```bash
# 分析 /data/片段 目录下所有视频
./run_full_tracker_analysis.sh
```

### 区域参数网格搜索

```bash
# 在单视频上搜索最优区域配置
./run_zone_grid_search.sh
```

## 📝 备注

1. **视频命名规范**: 格式为 `序号-数量头.mp4`，如 `1-46头.mp4` 表示该视频真实有 46 头猪
2. **轨迹过滤**: 生命周期过短（<10帧）的轨迹自动过滤，不计入最终统计
3. **场景特点**: 所有视频均为俯视角度的过道场景，猪单向通过
4. **误差来源**:
   - 多计：主要由于 ID 切换（同一目标被分配多个 ID）
   - 漏计：主要由于目标未完成完整的状态转换（如中途消失）
5. **可复现性**: 所有实验结果可通过重新运行脚本复现
